Що таке Git?

Git — популярна система контролю версій. Git був створений Лінусом Торвальдсом у 2005 році, і відтоді підтримується Джуніо Хамано.

Він використовується для:

  • Зміни коду відстеження
  • Відстеження того, хто вніс зміни
  • Співпраці в кодуванні

Що робить Git?

  • Керуйте проектами за допомогою репозиторіїв/сховищ
  • Clone/Клонуйте проект для роботи з локальною копією
  • Контролюйте та відстежуйте зміни за допомогою Staging/Постановки і Committing/Здійснення
  • Робіть Branch/Розгалуження та Merge/Злиття, щоб дозволити працювати над різними частинами та версіями проєкту
  • Pull/Витягніть останню версію проєкту до локальної копії
  • Push/Надішліть локальні оновлення в основний проєкт

Робота з Git

  • Ініціалізувати Git у теці, зробивши її Repository/Сховищем
  • Git тепер створює приховану теку для відстеження змін у цій теці
  • Коли файл змінено, додано або видалено, він вважається modified/зміненим
  • Ви вибираєте змінені файли, які хочете Stage/розмістити
  • Поставлені/Staged файли Committed/Здійсненні, що спонукає Git зберігати постійний знімок файлів
  • Git дозволяє переглядати повну історію кожного коміту
  • Ви можете повернутися до будь-якого попереднього коміту
  • Git не зберігає окрему копію кожного файлу в кожному коміті, але відстежує зміни, зроблені в кожному коміті!

Чому Git?

  • Понад 70% розробників використовують Git!
  • Розробники можуть працювати разом з будь-якої точки світу
  • Розробники можуть бачити повну історію проєкту
  • Розробники можуть повернутися до попередніх версій проєкту
