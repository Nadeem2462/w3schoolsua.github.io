Швидкий старт для GitHub Pages

Ви можете використовувати GitHub Pages, щоб продемонструвати деякі проєкти з відкритим кодом, розмістити блог або навіть поділитися своїм резюме. Цей посібник допоможе вам розпочати створення вашого наступного вебсайту.

GitHub Pages доступні у загальнодоступних репозиторіях із GitHub Free та GitHub Free для організацій, а також у загальнодоступних і приватних репозиторіях із GitHub Pro, GitHub Team, GitHub Enterprise Cloud та GitHub Enterprise Server. Для отримання додаткової інформації дивіться "GitHub Продукти."

Інтро

GitHub Pages – це загальнодоступні вебсторінки, розміщені та опубліковані через GitHub. Найшвидший спосіб розпочати роботу — скористатися інструментом вибору тем Jekyll для завантаження попередньо створеної теми. Потім ви можете змінити вміст і стиль своїх сторінок GitHub.

Цей посібник допоможе вам створити сайт користувача на username.github.io.

Створити свій вебсайт на GitHub Pages

  1. У верхньому правому куті будь-якої сторінки скористайтеся спадним меню та виберіть "New repository" ("Новий репозиторій").
  2. Введіть username.github.io як назву репозиторію. Замініть username своїм власним іменем користувача GitHub. Наприклад, якщо ваше ім’я користувача octocat, ім’я репозиторію має бути таким: octocat.github.io.
  3. Під назвою свого репозиторію клікніть меню "Settings" ("Налаштування").
  4. У розділі "Code and automation" ("Код і автоматизація") бічної панелі клікніть Pages.
  5. Клікніть "Choose a theme" ("Вибрати тему").
  6. Відкриється вікно вибору теми. Перегляньте доступні теми, а потім натисніть "Select theme" ("Вибрати тему"), щоб вибрати тему. Пізніше легко змінити тему, тож якщо ви не впевнені, просто виберіть одну із запропонованих.
  7. Після вибору теми файл README.md вашого репозиторію відкриється в редакторі файлів. У файлі README.md ви будете писати вміст для свого сайту. Ви можете відредагувати файл або залишити вміст за умовчанням.
  8. Завершивши редагування файлу, натисніть "Commit changes" ("Затвердити зміни").
  9. Відвідайте username.github.io, щоб переглянути свій новий вебсайт.
    Примітка: публікація змін на вашому сайті може тривати до 20 хвилин після того, як ви надішлете зміни на GitHub. Просто трохи зачекайте!

Зміна назви та опису

За замовчуванням назва вашого сайту – username.github.io. Ви можете змінити назву, відредагувавши файл _config.yml у своєму репозиторії. Ви також можете додати опис для свого сайту.

  1. Клікніть вкладку Code вашого репозиторію.
  2. У списку файлів клікніть _config.yml щоб відкрити файл.
  3. Клікніть щоб редагувати файл.
  4. Файл _config.yml уже містить рядок, який визначає тему для вашого сайту. Додайте новий рядок із title:, а потім потрібним заголовком. Додайте новий рядок із description:, а потім потрібним описом. Наприклад:
    5. theme: jekyll-theme-minimal
    title: Домашня веб-сторінка Octocat
    description: Додайте це в закладки, щоб стежити за оновленнями мого проєкту!
  5. Після завершення редагування файлу натисніть "Commit changes" ("Затвердити зміни").

Наступні кроки

Щоб отримати додаткові відомості про те, як додати додаткові сторінки на свій сайт, дивіться "Додавання вмісту на ваш сайт GitHub Pages за допомогою Jekyll."

Додаткову інформацію про налаштування сайту GitHub Pages за допомогою Jekyll дивіться в розділі "Про GitHub Pages і Jekyll."
